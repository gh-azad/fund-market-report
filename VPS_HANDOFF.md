# راهنمای تحویل سرور VPS - برای نشست‌های بعدی

> این فایل برای دستیارهای هوش مصنوعی (Claude) در نشست‌های بعدی نوشته شده تا بدون نیاز به توضیح مجدد کاربر، بدانند سرور چگونه ستاپ شده و چطور باید با آن کار کنند.

---

## خلاصه‌ی یک‌خطی
یک VPS ایرانی (Pars Pack) با کاربر `deploy` راه‌اندازی شده که به‌صورت خودکار داده‌های FipIran را جمع‌آوری، ذخیره و بکاپ می‌گیرد؛ همه‌چیز از طریق systemd timers بدون دخالت دستی اجرا می‌شود.

---

## 1. اطلاعات دسترسی به سرور

| مورد | مقدار |
|---|---|
| IP | `194.5.205.63` |
| کاربر SSH | `deploy` |
| احراز‌هویت | فقط کلید عمومی SSH (رمز عبور و root غیرفعال) |

| دامنه‌ی مرتبط | `agahreport.ir` (از طریق Cloudflare) |

> ⚠️ کلید SSH باید از قبل در اختیار نشست باشد یا کاربر باید آن را در دسترس قرار دهد. اگر کلید موجود نیست، از کاربر بخواهید آن را فراهم کند — این سرور از دسترسی رمز عبور پشتیبانی نمی‌کند.

## 2. چه چیزی روی این سرور اجرا می‌شود

### الف) Relay API
سرویسی که به‌جای کامپیوتر کاربر، داده‌های FipIran را می‌کشد و در دسترس قرار می‌دهد.

- آدرس: `https://relay.agahreport.ir`
- فچ خودکار: هر روز ساعت **۵:۰۰ UTC**
- ذخیره‌سازی: SQLite (فایل دیتابیس روی خود VPS)
- دسترسی به API: نیازمند توکن Bearer در هدر درخواست

**بررسی وضعیت سرویس:**
```bash
sudo systemctl status relay-api
sudo journalctl -u relay-api -n 50 --no-pager
```

### ب) Timerهای خودکار (systemd)
| Timer | وظیفه | زمان‌بندی |
|---|---|---|
| `relay-fetch.timer` | جمع‌آوری داده از FipIran | روزانه، ۵ صبح UTC |
| `relay-backup.timer` | بکاپ‌گیری محلی + GitHub | روزانه |

**بررسی زمان‌بندی و اجرای دستی:**
```bash
systemctl list-timers | grep relay
sudo systemctl start relay-fetch.service   # اجرای فوری دستی
```

### ج) بکاپ‌گیری
- **محلی:** نگه‌داری ۳۰ روز، روی خود VPS
- **برون‌سروری:** ریپوی خصوصی GitHub به نام `fund-market-backups`

**محل فایل‌های بکاپ محلی را از طریق SSH پیدا کنید** (مسیر دقیق را با `sudo systemctl cat relay-backup.service` یا بررسی crontab/سرویس تأیید کنید — این جزئیات باید در سرور مستند یا verify شوند، چون این فایل از روی خلاصه‌ی گفتگو بازسازی شده و مسیر فایل دقیق ذخیره نشده است).

## 3. کاری که هنوز ناقص است

**پایپ‌لاین اصلی صندوق‌ها هنوز روی VPS نصب نشده.**

این پایپ‌لاین (که فعلاً روی کامپیوتر شرکت اجرا می‌شود، نه روی VPS):
- صندوق‌های multi-NAV (مثل صنایع آگاه با زیرمجموعه‌های پترو/اتو/آلیاژ/بانکا) را مدیریت می‌کند
- مستقیماً از سایت‌های خود صندوق‌ها (نه FipIran) داده می‌کشد
- خروجی تولید می‌کند: `today.json`, `data.js`, و فایل‌های CSV تاریخی
- نتیجه را مستقیم به ریپوی گزارش (`fund-market-report` یا مشابه) push می‌کند

### قدم بعدی برای هر نشست جدید
1. از کاربر بپرس آیا اسکریپت‌های پایپ‌لاین را در ریپوی خصوصی جدیدی (`fund-market-pipeline`) گذاشته یا نه
2. اگر بله: ریپو را کلون کن، وابستگی‌ها را نصب کن، یک systemd timer جدید بساز که این پایپ‌لاین را روزانه اجرا کند
3. مطمئن شو که پایپ‌لاین بعد از اجرا، تغییرات را به ریپوی گزارش push می‌کند (نیاز به GitHub token با دسترسی write روی آن ریپو)

## 4. اصول کار با این سرور (برای دستیار هوش مصنوعی)

- **همیشه قبل از تغییر، وضعیت فعلی را با `systemctl status` و `journalctl` بررسی کن** — چون این سرور به‌صورت خودکار و بدون نظارت مداوم کاربر کار می‌کند.
- **قبل از هر ری‌استارت سرویس یا تغییر timer، هماهنگ کن** — چون این سرویس‌ها منبع داده‌ی زنده‌ی داشبورد هستند و قطعی می‌تواند به‌روزرسانی روزانه را از دست بدهد.
- **توکن‌ها و کلیدهای API را هرگز در ریپوی عمومی commit نکن.** هر متغیر حساس (GitHub token, API token) باید در فایل env روی خود VPS (مثلاً `/etc/relay-api/.env` یا مشابه) نگه‌داری شود؛ مسیر دقیق را با `sudo systemctl cat relay-api` روی سرور تأیید کن.
- **این فایل بازسازی خلاصه‌ی یک گفتگوی قبلی است، نه مستندات دقیق فنی سرور.** جزئیاتی مثل مسیر فایل‌ها، نام دقیق فایل‌های سرویس، و محتوای فایل env باید موقع اتصال به سرور مجدداً از طریق SSH تأیید شوند — روی درستی صددرصد این اعداد و مسیرها حساب نکن.

---

## 5. چک‌لیست شروع سریع برای نشست جدید

```bash
# 1. اتصال
ssh deploy@194.5.205.63

# 2. بررسی سلامت کلی
sudo systemctl status relay-api relay-fetch.timer relay-backup.timer

# 3. بررسی آخرین اجرای فچ و بکاپ
sudo journalctl -u relay-fetch -n 30 --no-pager
sudo journalctl -u relay-backup -n 30 --no-pager

# 4. تست API (نیاز به توکن)
curl -H "Authorization: Bearer <TOKEN>" https://relay.agahreport.ir/api/data
```

اگر همه‌چیز سبز بود، سرور سالم است و کار روی پایپ‌لاین صندوق‌ها را می‌توان شروع کرد.
