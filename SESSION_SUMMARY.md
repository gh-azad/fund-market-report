### 2. Relay API (`relay.agahreport.ir`)
**مقصد:** جایگزینی برای بکاپ‌گیری و دسترسی به داده‌های FipIran

**ویژگی‌ها:**
- ✅ اجرای خودکار هر روز ساعت **۵ صبح UTC** از سایت FipIran
- ✅ ذخیره‌ی داده‌ها در پایگاه‌ داده SQLite
- ✅ API محافظت‌شده با توکن احراز‌هویت
- ✅ HTTPS از طریق Cloudflare

**نقطه‌های پایانی (Endpoints):**
- `GET /api/data` - دریافت تمام داده‌های جمع‌آوری‌شده
- `GET /api/backup` - دریافت وضعیت آخرین بکاپ‌گیری
- نیازمند هدر `Authorization: Bearer <TOKEN>`

---

### 3. سیستم بکاپ‌گیری
**بکاپ محلی:**
- ✅ روزانه ساعت ۶ صبح UTC
- ✅ ذخیره‌ی ۳۰ روزه
- ✅ فایل‌های SQLite و لاگ‌ها

**بکاپ برون‌سروری:**
- ✅ ریپوی GitHub خصوصی: `fund-market-backups`
- ✅ اجرای خودکار روزانه
- ✅ احراز‌هویت با GitHub token

---

### 4. مدیریت سرور
**Systemd Services & Timers:**
- `relay-api.service` - Relay API
- `relay-backup.timer` - تایمر بکاپ‌گیری روزانه
- `relay-fetch.timer` - تایمر جمع‌آوری داده روزانه

**مشاهده‌ی وضعیت:**
```bash
sudo systemctl status relay-api
sudo systemctl status relay-fetch.timer
sudo systemctl status relay-backup.timer
```

## ⏳ کار باقی‌مانده

### 1. انتقال پایپ‌لاین صندوق‌ها
**وضعیت:** در انتظار اسکریپت‌های پایپ‌لاین

**مراحل:**
1. کاربر باید روی کامپیوتر شرکت اسکریپت‌های پایپ‌لاین را پیدا کند
2. این اسکریپت‌ها را در ریپوی خصوصی جدید GitHub (`fund-market-pipeline`) قرار دهد
3. من سپس:
   - اسکریپت‌ها را روی VPS نصب کنم
   - Systemd timer برای اجرای خودکار تنظیم کنم
   - آن‌ها را با Relay API یا GitHub actions ادغام کنم

**نکات مهم:**
- اسکریپت‌ها باید:
  - خودکار قابل اجرا باشند (بدون ورودی تعاملی)
  - نتیجه را در JSON/CSV ذخیره کنند
  - بتوانند هر روز چند بار اجرا شوند
  - لاگ‌های مناسبی تولید کنند
